## 🔄 核心流程

### 1. 授权生成流程

```
┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  Web界面 │────→│Controller│────→│ Service  │────→│ RSA签名  │
└─────────┘     └──────────┘     └──────────┘     └──────────┘
                                         │                │
                                         ↓                ↓
                                  ┌──────────┐     ┌──────────┐
                                  │ 保存到DB │     │返回Key   │
                                  └──────────┘     └──────────┘
```

**详细步骤：**

1. 用户在 Web 界面填写授权信息
2. 前端调用 `/api/license/generate` 接口
3. Controller 接收请求，调用 Service
4. Service 将 License 对象转为 JSON
5. 使用私钥对 JSON 数据进行 SHA256withRSA 签名
6. 将数据和签名组合，Base64 编码生成 License Key
7. 保存到数据库
8. 返回给前端

---

### 2. 授权验证流程

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│应用启动   │────→│读取License   │────→│解码Key       │
└──────────┘     └──────────────┘     └──────────────┘
                                              │
                                              ↓
                                     ┌──────────────┐
                                     │公钥验签      │
                                     └──────────────┘
                                              │
                                    ┌─────────┴─────────┐
                                    ↓                   ↓
                             ┌──────────┐       ┌──────────┐
                             │验证失败   │       │验证成功   │
                             │退出应用   │       │继续运行   │
                             └──────────┘       └──────────┘
                                                        │
                                                        ↓
                                               ┌──────────────┐
                                               │定时验证       │
                                               │(每小时)       │
                                               └──────────────┘
```

**详细步骤：**

1. 应用启动时，LicenseManager 初始化
2. 读取 `license/license.key` 文件
3. 使用 LicenseCodec 解码，分离数据和签名
4. 使用公钥验证签名（RSAUtil.verify）
5. 解析 JSON 得到 License 对象
6. 检查过期时间
7. 检查设备绑定（如果设置）
8. 验证通过，应用正常启动
9. 启动定时任务，每小时重新验证

## 🔐 安全架构

### RSA 密钥体系

```
┌─────────────────────────────────────────────┐
│           密钥对生成                          │
│                                             │
│  KeyPairGenerator (2048位)                  │
│         ↓                                   │
│  ┌──────────────┐    ┌──────────────┐      │
│  │   私钥        │    │   公钥        │      │
│  │ (服务器保管)  │    │ (分发给客户)  │      │
│  └──────────────┘    └──────────────┘      │
│         ↓                    ↓              │
│    用于签名              用于验签            │
└─────────────────────────────────────────────┘
```

### 授权码结构

```
License Key = Base64( JSON数据 + "." + 签名 )

JSON数据示例：
{
  "expireTime": "2026-12-31",
  "machineId": "ABC123",
  "type": "professional",
  "maxUsers": 100,
  "customerName": "测试客户"
}

签名 = SHA256withRSA(JSON数据, 私钥)
```

## 🗄️ 数据库设计

### license_record 表

```sql
字段说明：
├── id              : 主键，自增
├── license_key     : 授权码（TEXT）
├── expire_time     : 过期时间
├── machine_id      : 机器码（可选）
├── type            : 授权类型
├── max_users       : 最大用户数
├── modules         : 功能模块（JSON）
├── customer_name   : 客户名称
├── remark          : 备注
├── status          : 状态（0-禁用，1-启用）
├── deleted         : 逻辑删除（0-否，1-是）
├── create_time     : 创建时间
└── update_time     : 更新时间

索引：
├── PRIMARY KEY (id)
├── idx_customer_name (customer_name)
├── idx_expire_time (expire_time)
└── idx_status (status)
```
